Pushing local code to a new GitHub remote repository. Quick refresher!
Initialize local and remote repository

This is going to be a quick recap on how to work with local code that you want to push to a remote repository. We will talk about how we commit changes to edited files or stage new files. This guide assumes you have:

1. an existing GitHub account

2. have git installed and have setup your github authentication on the commandline using personal access tokens.

3. Have some kind of code project you want to create an online repo for on GitHub. If you have not setup working authentication yet, refer to the documentation for Personal Access Tokens on:

https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens

With these preliminaries out of the way, let’s get started. In our example we will have a directory that contains a project, let’s say /home/user1/dev/project1. This will be our current working directory. All commands will be performed inside the project directory, unless stated otherwise.

To initialize a local git repository say:

git init -b main

Where the -b flag is telling Git to start using branch main on the local instance.

Add all the currently untracked files in current code project directory by using the git add command:

git add .

Where the . symbol means add everything inside current directory, be mindful to include it.

To commit all files or changes use the following:

git commit -m ‘First commit’

where the -m flag is the Message sent with the commit. Since the comments might include whitespaces, you must put them in single or double quotations.

Before we push anything to any remote repository, we first have to create it. We can do that with the GitHub gh repo create subcommand and going through the interactive prompt.

When asked, select push local repository to remote

Hit enter to tell Git that the path is the current working directory (notice the . symbol).

Confirm default repository name (it will default to project folder name) or change as desired, hit enter.

Add optional description. Choose visibility default public or private as desired.

On the question 'Add a remote?' select yes. This will create our remote branch. What should the new remote be called? I use the default: origin.

Now we can use the git push command and you will see that git has enumerated, compressed and wrote all the objects to the upstream remote branch. When you look at your online GitHub, a new repo with the given name should now be created, and your commits should be visible as most recent in the given branch name.

Adding new files to the project

Most projects will have a readme.md file. This creates an opportunity to explain how to add new local files to the remote repository after creating them.

Create a new readme file in the local repository with: 

touch readme.md

You can also add some text to the readme if you want. Then use the following command:

git commit . -m ‘test’

You will see that the branch displays as up to date with origin/main, and nothing new was added to the commit, because the tracked files were not changed. We have to add the new readme.md file to the local repository file staging index for it to actually start tracking our new file.

First add our new file with: 

git add readme.md

We could have also use . again instead, to add changes from all files instead of just readme.md. Now create a git new commit with the command:

git commit . -m 'added a readme file'

Now perform another git push and you will see this time it has pushed the changed to the remote branch.

Deleting files from the project

If you deleted a file from the local repo and noticed it does not get deleted remote side, inside the local working directory perform command:

git rm readme.md

Notice that this is not a regular linux rm command, but prefixed with git, which means it is a git subcommand and removes files from the local git project index, not the actual file from disk.

After that use:

git commit . -m ‘Removed a specific file’

And then perform a git push to finally delete it remote. The opposite could also happen, where you already removed a file from the remote repository, and want to delete it locally. You can then simply perform a git pull inside current working directory, and then you will have the most recent version of the project with the file deleted.

This concludes my small tutorial on how to initialize a local git project repository and push it to a new remote.
